# OLCPixelAnimation-Test
simple extension test of OLCPixelGameEngine for animated sprites

Requires sprites in png format and only supports 1 single image for sprites in uniform cell size ie. 16x16, 16x32, 32x32.
Sprites can be played in reverse order if required by setting reverse boolean on object creation. (e.g. sprPlayer3 in test cpp)

Requires and extends OLCPixelGameEngine.h

Usage:
see demo it's pretty straight forward!
